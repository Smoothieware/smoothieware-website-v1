# {::nomarkdown}<gcode>G28.1</gcode>{:/nomarkdown} G-code

{::nomarkdown}
<review id="g28-1:version-specific-behavior">

{::nomarkdown}
<versioned orientation="vertical">
<v1>
{:/nomarkdown}

{::nomarkdown}<gcode>G28.1</gcode>{:/nomarkdown} means "store the origin/park position that the machine goes to when the {::nomarkdown}<gcode>G28</gcode>{:/nomarkdown} command is issued".

This position is stored into memory, but is lost if the machine is reset.

You can make that position permanent by sending the {::nomarkdown}<mcode>M500</mcode>{:/nomarkdown} G-code to save the position into the configuration override.

{::nomarkdown}
</v1>
<v2>
{:/nomarkdown}

In Smoothieware V2, {::nomarkdown}<gcode>G28.1</gcode>{:/nomarkdown} continues to support storing a park position, though its behavior in grbl mode may have been refined compared to V1. The command sets a predefined position that can be returned to using {::nomarkdown}<gcode>G28.2</gcode>{:/nomarkdown} in non-grbl mode.

Like V1, the position is stored in memory but is lost on reset. Use {::nomarkdown}<mcode>M500</mcode>{:/nomarkdown} to persist the position.

{::nomarkdown}
</v2>
</versioned>
</proposal>

## Format

The command is used as such:

```
G28.1 X10 Y10
```

Which means: record the origin position as being at the X position 10 and Y position 10

## Parameters

| Parameter | Usage                           | Example    |
| --------- | ------------------------------- | ---------- |
| `X`       | Record origin position for X axis | <gcode>G28.1</gcode> X10 |
| `Y`       | Record origin position for Y axis | <gcode>G28.1</gcode> Y10 |
| `Z`       | Record origin position for Z axis | <gcode>G28.1</gcode> Z10 |

This G-code sets the origin position that the {::nomarkdown}<gcode>G28</gcode>{:/nomarkdown} G-code goes to.

<v2>
{:/nomarkdown}

This G-code sets the park position. In non-grbl mode, {::nomarkdown}<gcode>G28.2</gcode>{:/nomarkdown} will move to the position set by {::nomarkdown}<gcode>G28.1</gcode>{:/nomarkdown}. In grbl mode, {::nomarkdown}<gcode>G28</gcode>{:/nomarkdown} performs a homing cycle instead.

Use the {::nomarkdown}<mcode>M500</mcode>{:/nomarkdown} G-code to save this position into persistent storage.

Related commands: {::nomarkdown}<gcode>G28</gcode>{:/nomarkdown} (home axes), {::nomarkdown}<gcode>G28.2</gcode>{:/nomarkdown} (go to park position), {::nomarkdown}<gcode>G28.3</gcode>{:/nomarkdown} (manual homing)
