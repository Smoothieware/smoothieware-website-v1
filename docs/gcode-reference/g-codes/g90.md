# {::nomarkdown}<gcode>G90</gcode>{:/nomarkdown} G-code

{::nomarkdown}
<a href="/images/absolute-coords.png">
  <img src="/images/absolute-coords.png" alt="Absolute Coordinates" style="width: 250px; height: 250px; float: right; margin-left: 1rem;"/>
</a>
{:/nomarkdown}

{::nomarkdown}<gcode>G90</gcode>{:/nomarkdown} means "all moves starting now are in absolute coordinates".

This means positions are given relative to the 0,0,0 point of the current workspace coordinate system, and not relative to the current position (which is what {::nomarkdown}<gcode>G91</gcode>{:/nomarkdown} does).

When Smoothie starts, {::nomarkdown}<gcode>G90</gcode>{:/nomarkdown} is the default mode.

However, it is common to still add the command at the beginning of G-code files.

{::nomarkdown}<gcode>G90</gcode>{:/nomarkdown}'s evil twin is {::nomarkdown}<gcode>G91</gcode>{:/nomarkdown}.

## Format

{::nomarkdown}
<review id="g90:format-section">
<proposal>
{:/nomarkdown}

The command is used as such:

{::nomarkdown}
<versioned orientation="vertical">
<v1>
{:/nomarkdown}

```
G90
```

Which means: starting now all positions are absolute.

{::nomarkdown}
</v1>
<v2>
{:/nomarkdown}

```
G90        ; Standard G90 (subcode 0, default)
G90.0      ; Explicit subcode notation (equivalent to G90)
```

Which means: starting now all positions are absolute. V2 supports explicit subcode notation (G90.0) but only subcode 0 is valid.

{::nomarkdown}
</v2>
</versioned>
{:/nomarkdown}

{::nomarkdown}
</proposal>
<original>
{:/nomarkdown}

```

```

Which means: starting now all positions are absolute.

## Subcode

{::nomarkdown}
<versioned orientation="vertical">
<v1>
{:/nomarkdown}

V1 does not support subcodes. {::nomarkdown}<gcode>G90</gcode>{:/nomarkdown} is always interpreted as absolute mode.

{::nomarkdown}
</v1>
<v2>
{:/nomarkdown}

**V2 Subcode Support:**

- **0**: Standard absolute positioning mode (default if no subcode specified)
- **Other values**: Not supported; returns an error if specified

This design allows for future extensions (e.g., G90.1, G90.2) while maintaining compatibility with existing code.

{::nomarkdown}
</v2>
</versioned>
{:/nomarkdown}

## Implementation Details

{::nomarkdown}
<review id="g90:implementation-section">
<proposal>
{:/nomarkdown}

{::nomarkdown}
<versioned orientation="vertical">
<v1>
{:/nomarkdown}

**V1 Implementation:**

- Sets both `absolute_mode` and `e_absolute_mode` to true
- Affects X, Y, Z axes AND extruder (E axis)
- Coordinates are interpreted as absolute positions in the current workspace coordinate system
- Workspace offsets (G54-G59, G92, tool offset) are still applied
- Modal command (remains active until changed by G91 or M83)

{::nomarkdown}
</v1>
<v2>
{:/nomarkdown}

**V2 Implementation:**

- Sets both `absolute_mode` and `e_absolute_mode` to true (same as V1)
- Includes subcode validation: only subcode 0 is accepted
- Returns an error for invalid subcodes instead of silently accepting them
- Affects X, Y, Z axes AND extruder (E axis)
- Coordinates are interpreted as absolute positions in the current workspace coordinate system
- Workspace offsets (G54-G59, G92, tool offset) are still applied
- Modal command (remains active until changed by G91 or M83)

{::nomarkdown}
</v2>
</versioned>
{:/nomarkdown}
