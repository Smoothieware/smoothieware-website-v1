# <gcode>G0</gcode> G-code
{:/nomarkdown}

<gcode>G0</gcode> means "move without activating the tool".

On CNC mills, it is used to go to a new area without cutting, it is also called a "rapid" move. 

On laser cutters, it is used the same way, and the laser is automatically turned off during the move.

On 3D printers, it is also called a "travel" move.

{::nomarkdown}
While traditionally on CNC mills <gcode>G0</gcode> is supposed to be an "un-coordinated" move, on Smoothie it is coordinated anyway.

<gcode>G0</gcode>'s evil twin is <gcode>G1</gcode>, which means "move while using the tool".
{:/nomarkdown}

## Format

The command is used as such:

```
G0 X10 Y20 F30
```

Which means: move (without cutting) to X position 10, Y position 20, at a speed of 30 millimeters/minute.

## Parameters

| Parameter | Usage | Example |
| --------- | ----- | ------- |
| `X` | Move to this position in the X axis | <gcode>G0</gcode> X10 |
| `Y` | Move to this position in the Y axis | <gcode>G0</gcode> Y10 |
| `Z` | Move to this position in the Z axis | <gcode>G0</gcode> Z10 |
| `A` | Move to this position in the A axis | <gcode>G0</gcode> A10 |
| `B` | Move to this position in the B axis | <gcode>G0</gcode> B10 |
| `C` | Move to this position in the C axis | <gcode>G0</gcode> C10 |
| `F` | Move at this speed in millimeters/minute | <gcode>G0</gcode> Z10 F100 |

## Feedrate

{::nomarkdown}
The `F` parameter is modal. The speed is remembered and each subsequent command uses it.
{:/nomarkdown}

This means if you do:

```
G0 X10 F100
G0 X20
G0 X30 F200
G0 X40
```

The first two moves will happen at 100 mm/minute, and the last two moves will happen at 200mm/minute.

<sl-alert variant="primary" open>
  <sl-icon slot="icon" name="lightbulb"></sl-icon>
  <strong>Independent feedrates</strong><br><br>
  Note that both <gcode>G0</gcode> and <gcode>G1</gcode> have their own modal feedrates, and setting one doesn't influence the other.<br><br>
  This means if you do:<br><br>
  <code><gcode>G0</gcode> X10 F100<br>
  <gcode>G1</gcode> X20 F200<br>
  <gcode>G0</gcode> X30</code><br><br>
  The third move will actually move at 100 mm/minute as that is the latest feedrate set for <gcode>G0</gcode> moves.
</sl-alert>

## Default feedrate

{::nomarkdown}
If you have just started your Smoothieboard and have never sent an `F` parameter, your <gcode>G0</gcode> moves will move at the default "seek" feedrate.
{:/nomarkdown}

This is set in the [configuration file](configuring-smoothie) by setting the <setting v1="default_seek_rate" v2="motion control.default_seek_rate"></setting> config option.

## Modal use

{::nomarkdown}
The <gcode>G0</gcode> command is modal, if a line doesn't contain any gcode, it is assumed to be a <gcode>G0</gcode> or a <gcode>G1</gcode> command depending on the latest used.
{:/nomarkdown}

This means if you do:

```
G0 X10
 X20
 Y10
```

It is the same as doing:

```
G0 X10
G0 X20
G0 Y10
```

{::nomarkdown}
Note the space character before each modal line, without that space, Smoothie will not know this is a modal command.
{:/nomarkdown}

## Absolute and relative motion

{::nomarkdown}
There are two ways to specify positions: absolutely or relatively.

Smoothie is in absolute mode by default.

You enter absolute mode by sending the <gcode>G90</gcode> Gcode, and you enter the relative mode by sending the <gcode>G91</gcode> Gcode.

In absolute mode, all positions passed to <gcode>G0</gcode> are relative to the 0 position for the current work coordinate.

In relative mode, all positions passed to <gcode>G0</gcode> are relative to the current position.
{:/nomarkdown}

This means if you do:

```
G90
G0 X10
G0 X20
```

{::nomarkdown}
Smoothie will move to the X position 10, then move to the X position 20, relative to the origin.

But if you do:
{:/nomarkdown}

```
G91
G0 X10
G0 X20
```

{::nomarkdown}
Smoothie will move 10 millimeters relative to the current position, then move 20 millimeters further away from that new position (moving 30 millimeters total).
{:/nomarkdown}
